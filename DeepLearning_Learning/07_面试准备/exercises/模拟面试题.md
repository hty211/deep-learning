### 1.1 神经网络

**Q1: 解释梯度消失和梯度爆炸的原因及解决方案**

<details>
<summary>参考答案</summary>

```
原因:
- 梯度消失: 链式法则中多个小于1的梯度相乘
- 梯度爆炸: 链式法则中多个大于1的梯度相乘

解决方案:
1. 梯度消失:
   - 使用ReLU等非饱和激活函数
   - 残差连接
   - Batch/Layer Normalization
   - 合适的权重初始化(He/Xavier)

2. 梯度爆炸:
   - 梯度裁剪
   - 合适的权重初始化
   - Batch Normalization
```
</details>

### 1.2 优化算法

**Q3: Adam优化器的原理？为什么常用？**

<details>
<summary>参考答案</summary>

```
原理:
- 结合Momentum和RMSprop
- 维护一阶矩估计m和二阶矩估计v
- 偏差修正

公式:
m_t = β₁m_{t-1} + (1-β₁)g_t
v_t = β₂v_{t-1} + (1-β₂)g_t²
m̂_t = m_t / (1-β₁^t)
v̂_t = v_t / (1-β₂^t)
w_t = w_{t-1} - η * m̂_t / (√v̂_t + ε)

为什么常用:
- 自适应学习率
- 对超参数不敏感
- 收敛快
- 适合大多数任务
```
</details>

### 2.1 词向量

**Q4: Word2Vec的负采样是什么？为什么有效？**

<details>
<summary>参考答案</summary>

```
负采样:
- 将多分类问题转为二分类
- 正样本: 中心词和真实上下文词
- 负样本: 中心词和随机采样词

目标函数:
L = log(σ(v'_c · v_w)) + Σ log(σ(-v'_c · v_n))

为什么有效:
1. 计算效率: 不需要计算整个词表的softmax
2. 效果好: 区分正负样本学习词向量
3. 稀疏更新: 只更新正负样本对应的词向量
```
</details>

### 2.2 Transformer

**Q5: Transformer相比RNN的优势？**

<details>
<summary>参考答案</summary>

```
1. 并行计算
   - RNN必须顺序计算
   - Transformer可以并行处理所有位置

2. 长距离依赖
   - RNN需要多步传递信息
   - Transformer任意位置直接交互

3. 梯度传播
   - RNN容易梯度消失
   - Transformer梯度路径更短

4. 灵活性
   - 可变长度处理更自然
   - 易于扩展和修改
```
</details>

**Q6: 解释Multi-Head Attention的计算过程**

<details>
<summary>参考答案</summary>

```
步骤:
1. 线性变换: Q, K, V分别投影到h个子空间
   Q_i = QW_i^Q, K_i = KW_i^K, V_i = VW_i^V

2. 各头计算注意力:
   head_i = Attention(Q_i, K_i, V_i)
         = softmax(Q_i K_i^T / √d_k) V_i

3. 拼接所有头:
   MultiHead = Concat(head_1, ..., head_h)

4. 最终线性变换:
   Output = MultiHead W^O

参数量:
- 每个头维度: d_model / h
- 总参数量不变
```
</details>

```
   - 学习正确表示
```

<details>
<summary>参考答案</summary>

```python
import torch
import math

def self_attention(Q, K, V):
    d_k = Q.size(-1)
    
    # 计算注意力分数
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
    
    # Softmax归一化
    attention_weights = torch.softmax(scores, dim=-1)
    
    # 加权求和
    output = torch.matmul(attention_weights, V)
    
    return output, attention_weights
```

```python
def beam_search(model, initial_input, beam_width, max_length):
    # 初始化beam
    beams = [(initial_input, 0)]  # (sequence, log_prob)
    completed = []
    
    for _ in range(max_length):
        new_beams = []
        
        for seq, score in beams:
            # 获取模型输出
            with torch.no_grad():
                output = model(seq)
                probs = torch.log_softmax(output[:, -1, :], dim=-1)
            
            # 获取top-k候选
            top_probs, top_indices = probs.topk(beam_width)
            
            for prob, idx in zip(top_probs[0], top_indices[0]):
                new_seq = torch.cat([seq, idx.unsqueeze(0).unsqueeze(0)], dim=1)
                new_score = score + prob.item()
                
                # 检查是否结束
                if idx.item() == eos_token_id:
                    completed.append((new_seq, new_score))
                else:
                    new_beams.append((new_seq, new_score))
        
        # 保留top beam_width个候选
        beams = sorted(new_beams, key=lambda x: x[1], reverse=True)[:beam_width]
    
    # 返回最佳序列
    completed.extend(beams)
    best = max(completed, key=lambda x: x[1] / len(x[0][0]))
    return best[0]
```
