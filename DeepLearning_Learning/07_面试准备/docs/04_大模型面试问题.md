# 大模型面试问题

## 1. Transformer架构

### 1.1 自注意力机制

**Q: 解释自注意力机制的计算过程**

```
输入: X ∈ R^(n×d)

步骤:
1. 线性变换得到Q, K, V
   Q = XW_Q, K = XW_K, V = XW_V

2. 计算注意力分数
   Scores = QK^T / √d_k

3. Softmax归一化
   Attention = softmax(Scores)

4. 加权求和
   Output = Attention × V

复杂度分析:
- 时间复杂度: O(n²d)
- 空间复杂度: O(n²)
- n是序列长度，d是维度

为什么除以√d_k:
- 防止点积过大
- 使softmax梯度更稳定
- 推导: 当d_k很大时，点积方差为d_k
```

### 1.2 多头注意力

**Q: 为什么使用多头注意力？**

```
原理:
- 将d维空间分成h个子空间
- 每个头学习不同的表示
- 最后拼接并线性变换

计算过程:
1. 分头: Q_i = QW_i^Q, K_i = KW_i^K, V_i = VW_i^V
2. 各头计算: head_i = Attention(Q_i, K_i, V_i)
3. 拼接: MultiHead = Concat(head_1,...,head_h)W^O

优势:
- 捕获不同类型的依赖关系
- 每个头关注不同的语义信息
- 增强模型表达能力

经验:
- 通常使用8或16个头
- 头数过多可能导致信息冗余
```

### 1.3 位置编码

**Q: 为什么需要位置编码？Transformer如何编码位置？**

```
为什么需要:
- 自注意力是置换不变的
- 无法区分不同位置
- 需要额外注入位置信息

正弦位置编码:
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

特点:
1. 固定编码，不需要学习
2. 可以外推到更长序列
3. 相对位置可以通过线性变换得到

可学习位置编码:
- 随机初始化，训练学习
- 更灵活，但有长度限制
- BERT使用这种方式

相对位置编码:
- 编码相对位置关系
- T5、DeBERTa使用
```

## 2. 预训练模型

### 2.1 BERT系列

**Q: BERT的优缺点？有哪些改进版本？**

```
优点:
- 双向上下文理解
- 预训练+微调范式
- 多任务适应能力强

缺点:
- [MASK]在预训练和微调时不一致
- NSP任务效果存疑
- 生成能力弱

改进版本:

1. RoBERTa
   - 更大数据集和训练时间
   - 去掉NSP任务
   - 动态遮蔽

2. ALBERT
   - 参数共享减少参数量
   - 因式分解嵌入
   - 句子顺序预测替代NSP

3. SpanBERT
   - 随机遮蔽连续片段
   - 去掉NSP
   - 更好地学习短语表示

4. ELECTRA
   - 替换词检测任务
   - 更高效的预训练
   - 所有token都参与训练
```

```
GPT-1 (2018):

GPT-2 (2019):

GPT-3 (2020):

关键创新:

```

   - 动态损失缩放

## 4. 参数高效微调

### 4.1 LoRA

**Q: LoRA的原理和优势？**

```
原理:
- 权重更新ΔW可以分解为低秩矩阵
- W' = W + BA
- B ∈ R^(d×r), A ∈ R^(r×k), r << min(d,k)

优势:
1. 参数量少
   - 只训练r×(d+k)参数
   - 通常<1%原始参数

2. 无推理延迟
   - 可合并到原权重
   - W_new = W + BA

3. 易于切换
   - 保存多个LoRA适配器
   - 快速切换不同任务

配置建议:
- Rank: 8-64
- Alpha: 通常2×rank
- Target: q_proj, v_proj开始
```

### 4.2 其他PEFT方法

**Q: 除了LoRA还有哪些参数高效微调方法？**

```
1. Adapter
   - 在层间插入小型网络
   - 瓶颈结构: d -> r -> d
   - 增加推理延迟

2. Prefix Tuning
   - 在输入前添加可学习前缀
   - 每层都有独立前缀
   - 占用序列长度

3. Prompt Tuning
   - 只在输入层添加虚拟token
   - 参数最少
   - 效果可能不如其他方法

4. BitFit
   - 只微调bias参数
   - 极简方法
   - 适合简单任务

方法选择:
- 追求效果: LoRA
- 追求效率: Prompt Tuning
- 灵活性: Adapter
```

## 5. 模型量化

**Q: 模型量化的方法？**

```
量化类型:

1. 训练后量化(PTQ)
   - 训练完成后量化
   - 不需要重新训练
   - 简单快速

2. 量化感知训练(QAT)
   - 训练时模拟量化
   - 精度损失更小
   - 需要重新训练

量化精度:
- INT8: 25%大小，精度损失小
- INT4: 12.5%大小，精度损失中等
- NF4: 专为正态分布权重设计

GPTQ:
- 训练后量化方法
- 基于Hessian信息
- 适合大语言模型

QLoRA:
- 4-bit量化 + LoRA微调
- 大幅降低显存需求
- 保持全精度微调效果
```

## 6. RAG技术

**Q: RAG的原理和优势？**

```
原理:
1. 检索: 从知识库检索相关文档
2. 增强: 将检索结果作为上下文
3. 生成: 基于增强上下文生成回答

优势:
1. 知识可更新
   - 不需要重新训练
   - 实时更新知识库

2. 可解释性
   - 提供答案来源
   - 可追溯引用

3. 减少幻觉
   - 基于检索内容生成
   - 提高事实准确性

4. 领域适配
   - 注入领域知识
   - 专业问答

关键组件:
- 文档处理: 分块、加载
- 嵌入模型: 文本向量化
- 向量数据库: 存储、检索
- 大语言模型: 生成回答
```

```
1. 零样本提示

5. ReAct
   推理+行动交替
   Thought -> Action -> Observation

6. 角色扮演
   "你是一个专业的..."

7. 结构化输出
   "请以JSON格式输出..."
```

## 8. 对齐技术

**Q: 什么是RLHF？**

```
RLHF (Reinforcement Learning from Human Feedback):

步骤:
1. 监督微调(SFT)
   - 用人类示范数据微调
   - 学习基本对话能力

2. 奖励模型训练(RM)
   - 人类对模型输出排序
   - 训练奖励模型预测偏好

3. 强化学习优化(PPO)
   - 用奖励模型作为奖励信号
   - 优化模型策略

目的:
- 使模型行为符合人类偏好
- 减少有害输出
- 提高有用性和安全性

替代方法:
- DPO: 直接偏好优化
- 简化RLHF流程
- 不需要训练奖励模型
```
